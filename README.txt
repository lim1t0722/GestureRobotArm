1. 测试环境要求与配置说明
===============================================================================

1.1 硬件环境
  ┌──────────────┬────────────────────────────────────────────────┐
  │ 项目         │ 要求                                          │
  ├──────────────┼────────────────────────────────────────────────┤
  │ 主控板       │ STM32F103C8T6最小系统板(Blue Pill)            │
  │ 调试器       │ ST-Link V2 (SWD接口)                          │
  │ 姿态传感器   │ GY-521 MPU6050模块                            │
  │ PWM驱动      │ PCA9685 16路舵机驱动板                        │
  │ 舵机         │ MG996R ×3 (底座/大臂/小臂)                    │
  │ 旋钮         │ 10KΩ电位器(可选,未连接时自动检测)             │
  │ 按键         │ 轻触按键(接PA0)                               │
  │ LED          │ 板载LED(PC13)或外接LED+1KΩ电阻               │
  │ 电源         │ 5V/6V独立电源(≥3A)供舵机 + 3.3V供主控        │
  │ 串口工具     │ USB-TTL转换器(如CH340)                        │
  └──────────────┴────────────────────────────────────────────────┘

1.2 软件环境
  ┌──────────────┬────────────────────────────────────────────────┐
  │ 项目         │ 要求                                          │
  ├──────────────┼────────────────────────────────────────────────┤
  │ IDE          │ STM32CubeIDE 1.14+                            │
  │ 工具链       │ arm-none-eabi-gcc (IDE内置)                   │
  │ 烧录工具     │ STM32CubeProgrammer 或 IDE内置烧录            │
  │ 串口终端     │ 串口助手/SSCOM/PuTTY (115200,8N1)             │
  └──────────────┴────────────────────────────────────────────────┘

1.3 编译配置验证
  步骤1: 打开STM32CubeIDE, 导入项目
  步骤2: 右键项目 → Properties → C/C++ Build → Settings → MCU Settings
  步骤3: 确认以下配置:
    - MCU: STM32F103C8Tx
    - Floating-point unit: None
    - Floating-point ABI: Soft
  步骤4: Project → Build All, 确认编译无错误

1.4 接线检查清单(上电前必查)
  □ STM32 PB6 ↔ MPU6050 SCL (4.7K上拉至3.3V)
  □ STM32 PB7 ↔ MPU6050 SDA (4.7K上拉至3.3V)
  □ STM32 PB6 ↔ PCA9685 SCL (共用I2C总线)
  □ STM32 PB7 ↔ PCA9685 SDA (共用I2C总线)
  □ STM32 PA0 ↔ 按键 → GND
  □ STM32 PA1 ↔ 旋钮中间抽头(可选)
  □ STM32 PC13 ↔ LED阴极(低电平点亮)
  □ PCA9685 CH0 ↔ 底座舵机信号线
  □ PCA9685 CH1 ↔ 大臂舵机信号线
  □ PCA9685 CH2 ↔ 小臂舵机信号线
  □ 舵机VCC ↔ 独立5V/6V电源(非3.3V!)
  □ 所有GND共地连接
  □ MPU6050 AD0 ↔ GND (地址0x68)
  □ PCA9685 A0-A5 ↔ GND/悬空 (地址0x40)

2. 系统时钟模块测试
===============================================================================

2.1 测试范围
  - HSE外部晶振启动
  - PLL倍频配置(HSE×9=72MHz)
  - 总线分频(AHB/APB1/APB2)
  - Flash等待周期

2.2 测试目标
  验证系统时钟正确配置为72MHz, 各总线频率符合设计要求。

2.3 测试步骤
  步骤1: 编译并烧录程序
  步骤2: 连接ST-Link, 进入调试模式
  步骤3: 在SystemInit()函数末尾设置断点
  步骤4: 检查寄存器值:
    - RCC->CR: HSERDY=1, PLLRDY=1
    - RCC->CFGR: SWS=0x08(PLL作为系统时钟)
    - FLASH->ACR: LATENCY=2
  步骤5: 在Watch窗口查看SystemCoreClock变量

2.4 预期结果与判定标准
  ┌──────────────────────┬────────────────────────────────┐
  │ 检查项               │ 预期值                          │
  ├──────────────────────┼────────────────────────────────┤
  │ SystemCoreClock      │ 72000000                        │
  │ SWS位                │ 0x08 (PLL选中)                  │
  │ HSERDY位             │ 1 (HSE就绪)                     │
  │ PLLRDY位             │ 1 (PLL就绪)                     │
  │ Flash LATENCY        │ 2 (2个等待周期)                 │
  └──────────────────────┴────────────────────────────────┘
  判定: 全部符合 → PASS, 任一不符 → FAIL

4. I2C通信模块测试
===============================================================================

4.1 测试范围
  - I2C1初始化(400kHz快速模式)
  - MPU6050通信(地址0xD0)
  - PCA9685通信(地址0x80)
  - I2C BUSY标志自动清除(V2.0新增)
  - 安全I2C读写封装(V2.0新增)
  - 连续错误计数与总线恢复(V2.0新增)

4.2 测试目标
  验证I2C总线能与两个从设备正常通信, 无总线挂死, BUSY标志异常能自动恢复。

4.3 测试步骤
  步骤1: 烧录程序, 连接串口终端(115200,8N1)
  步骤2: 观察串口输出:
    - "Gesture Robot Arm - System Start"
    - "SYSCLK=72000000Hz"
    - "[INIT] PCA9685 OK"
    - "[INIT] MPU6050 OK"
  步骤3: 若PCA9685初始化失败, LED快闪3次/秒
  步骤4: 若MPU6050初始化失败, LED中速闪2次/秒
  步骤5: 用逻辑分析仪抓取I2C波形:
    - 在MPU6050初始化期间, 应看到对0xD0的写操作
    - 在PCA9685初始化期间, 应看到对0x80的写操作
  步骤6: 运行1分钟后检查串口诊断输出:
    - "[I2C] err=0 recv=0" 表示无I2C错误
    - 如果err>0, 说明存在I2C通信问题

4.4 预期结果与判定标准
  ┌──────────────────────┬────────────────────────────────┐
  │ 检查项               │ 预期值                          │
  ├──────────────────────┼────────────────────────────────┤
  │ 串口输出             │ 初始化OK消息依次出现            │
  │ I2C时钟频率          │ 400kHz ±5%                     │
  │ MPU6050应答          │ ACK(无NACK)                     │
  │ PCA9685应答          │ ACK(无NACK)                     │
  │ 总线挂死             │ 无(自动BUSY标志清除)            │
  │ I2C错误计数          │ 运行10分钟err<5                 │
  └──────────────────────┴────────────────────────────────┘

4.5 常见问题
  - I2C总线挂死: 系统自动检测BUSY标志并清除,无需手动干预
  - MPU6050无ACK: 检查AD0引脚接地, 确认模块供电3.3V
  - PCA9685无ACK: 检查A0-A5地址引脚, 确认OE引脚悬空或接低
  - I2C错误频繁: 检查上拉电阻(4.7K), 确认SDA/SCL无短路

5. MPU6050姿态传感器模块测试
===============================================================================

5.1 测试范围
  - WHO_AM_I设备ID读取
  - 陀螺仪零偏校准(200次采样)
  - 加速度计/陀螺仪原始数据读取
  - 互补滤波角度计算(alpha=0.96)
  - 角度限幅(±180°)
  - 数据有效性验证(V2.0新增)
  - 连续错误计数与自动恢复(V2.0新增)
  - 初始化后角度预计算(V2.0新增)

5.2 测试目标
  验证MPU6050能正确输出姿态角, 互补滤波响应平滑, 角度限幅有效, 数据验证可靠。

5.3 测试步骤
  步骤1: 烧录程序, 保持MPU6050水平静止
  步骤2: 等待校准完成(约2秒, 200次采样+200ms稳定等待)
  步骤3: 观察串口诊断输出(每秒1次):
    - "[MPU6050] R=0.0 P=0.0 Y=0.0 err=0/XX OK"
  步骤4: 静止时观察串口输出:
    - Roll应接近0°, Pitch应接近0°
    - 数值波动应<±3°(死区3°内不更新)
  步骤5: 向左倾斜传感器:
    - Roll应为负值(约-90°)
  步骤6: 向右倾斜传感器:
    - Roll应为正值(约+90°)
  步骤7: 向前倾斜传感器:
    - Pitch应为正值
  步骤8: 快速旋转后静止:
    - 角度应在1-2秒内稳定
    - Yaw不应超出±180°
  步骤9: 检查数据有效性:
    - err计数应为0或极低
    - 状态应显示"OK"而非"ERR"

5.4 预期结果与判定标准
  ┌──────────────────────┬────────────────────────────────┐
  │ 检查项               │ 预期值                          │
  ├──────────────────────┼────────────────────────────────┤
  │ 静态Roll/Pitch       │ 0° ±3°                          │
  │ 静态波动             │ <±3°(死区内)                    │
  │ Roll响应范围         │ -90° ~ +90°                     │
  │ Pitch响应范围        │ -90° ~ +90°                     │
  │ Yaw限幅              │ 不超出±180°                     │
  │ 响应延迟             │ <300ms                          │
  │ 校准后漂移           │ <1°/分钟                        │
  │ 数据有效性           │ 状态显示OK                      │
  │ I2C错误率            │ <1%                             │
  └──────────────────────┴────────────────────────────────┘

5.5 常见问题
  - 角度持续漂移: 重新上电校准, 确保校准时传感器静止
  - 角度跳变: 检查I2C连接稳定性, 确认400kHz信号完整性
  - 角度不变化: 检查MPU6050 SCL/SDA是否接反, 查看串口err计数
  - err计数持续增长: I2C通信故障, 检查接线和上拉电阻
  - 状态显示ERR: 数据验证失败, 检查MPU6050供电和连接

6. PCA9685 PWM驱动模块测试
===============================================================================

6.1 测试范围
  - PCA9685初始化(50Hz PWM)
  - PWM通道输出
  - 角度-脉宽映射(0°~180° → 0.5ms~2.5ms)
  - 浮点角度控制
  - 安全I2C通信(V2.0新增)

6.2 测试目标
  验证PCA9685能正确输出50Hz PWM信号, 角度映射准确。

6.3 测试步骤
  步骤1: 烧录程序, 连接一个舵机到CH0
  步骤2: 上电后舵机应转到90°中位
  步骤3: 用示波器测量CH0信号:
    - 周期: 20ms (50Hz)
    - 中位脉宽: 约1.5ms
  步骤4: 临时修改测试代码, 依次测试0°/90°/180°:
    PCA9685_SetServoAngle(0, 0);   HAL_Delay(1000);
    PCA9685_SetServoAngle(0, 90);  HAL_Delay(1000);
    PCA9685_SetServoAngle(0, 180); HAL_Delay(1000);
  步骤5: 观察舵机位置:
    - 0°: 舵机转到一端极限
    - 90°: 舵机转到中位
    - 180°: 舵机转到另一端极限
  步骤6: 测量各角度脉宽:
    - 0°: 约0.5ms
    - 90°: 约1.5ms
    - 180°: 约2.5ms

6.4 预期结果与判定标准
  ┌──────────────────────┬────────────────────────────────┐
  │ 检查项               │ 预期值                          │
  ├───────────────┼────────────────────────────────┤
  │ PWM频率              │ 50Hz ±1Hz                       │
  │ 0°脉宽               │ 0.5ms ±0.05ms                   │
  │ 90°脉宽              │ 1.5ms ±0.05ms                   │
  │ 180°脉宽             │ 2.5ms ±0.05ms                   │
  │ 舵机运动             │ 平滑无抖动                      │
  │ CH0/CH1/CH2独立控制  │ 各通道互不干扰                  │
  └──────────────────────┴────────────────────────────────┘

6.5 常见问题
  - 舵机抖动: 检查供电是否充足(≥5V/2A), 确认GND共地
  - 舵机不转: 检查信号线连接, 确认PCA9685 OE引脚为低
  - PWM频率偏移: 检查PCA9685外部晶振(25MHz), 确认prescale=121

  - ADC1通道1初始化
  - 旋钮ADC值读取

  - 浮空输入自动检测(V2.0新增)
  - IIR低通滤波(V2.0新增)

7.2 测试目标
  验证ADC能正确读取旋钮位置, 超时保护有效, 浮空输入时自动返回安全默认值。

7.3 测试步骤

  步骤3: 旋钮左旋到底: ADC值应接近0
  步骤4: 旋钮右旋到底: ADC值应接近4095
  步骤5: 旋钮中位: ADC值应接近2048
  步骤6: 缓慢旋转旋钮: ADC值应连续变化无跳变
  步骤7: 断开PA1连接(模拟浮空输入):
    - 系统不应死锁
    - ADC_ReadKnob应返回2048(安全默认值)
    - 底座舵机应保持90°中位不动
    - 串口显示"knob=NO"

8. 机械臂舵机控制模块测试
===============================================================================

8.1 测试范围
  - 手势模式角度映射
  - 手动模式旋钮+按键控制
  - 多级位置控制(3级)
  - 平滑运动算法
  - 角度限位保护
  - 底座舵机速率限制(V2.0新增)
  - 手势死区增大至3°(V2.0新增)
  - 旋钮连接状态检测(V2.0新增)

8.2 测试目标
  验证机械臂在手势/手动两种模式下响应正确, 运动平滑, 限位有效, 底座无异常转动。

8.3 测试步骤 — 手势模式
  步骤1: 上电默认进入手势模式(LED慢闪)
  步骤2: 传感器水平放置: 机械臂应在90°中位
  步骤3: 向左倾斜传感器: 底座舵机应向左旋转(平滑,1°/步)
  步骤4: 向右倾斜传感器: 底座舵机应向右旋转(平滑,1°/步)
  步骤5: 向前倾斜传感器: 大臂应前倾, 小臂联动
  步骤6: 快速倾斜后静止: 机械臂应平滑跟随, 无突变
  步骤7: 微小倾斜(<3°): 舵机不应响应(死区保护)
  步骤8: 观察串口诊断输出:
    - "[ARM] Tgt: B=XX.X Big=XX.X S=XX.X Cur: B=XX.X Big=XX.X S=XX.X GES"

8.4 测试步骤 — 手动模式
  步骤1: 长按按键>1秒, 切换到手动模式(LED快闪, 闪4次确认)
  步骤2: 旋转旋钮: 底座舵机应跟随旋转(0°~180°)
  步骤3: 未接旋钮时: 底座应保持90°中位不动
  步骤4: 短按按键: 大臂/小臂切换位置级别
    - 第1次按: 中位(90°,90°)
    - 第2次按: 半抬(60°,120°)
    - 第3次按: 全抬(30°,160°)
    - 第4次按: 回到中位(循环)
  步骤5: 观察每次切换: 舵机应平滑过渡, 非瞬间跳变
  步骤6: 长按按键>1秒: 切回手势模式(LED闪2次确认)

8.5 预期结果与判定标准

13. V2.0问题诊断与修复记录
===============================================================================

13.1 问题描述
  编译成功烧录上电后出现以下异常:
  1. 三个舵机上电后能完成中立位复位,但后续无正常操作响应
  2. 底座舵机偶尔出现无规律转动
  3. 手动移动MPU6050时,舵机未能按预期进行相应转动

13.2 根因分析

  问题1: 舵机初始化后无响应
  ┌─────────────────────────────────────────────────────────────────────┐
  │ 根因: STM32F1 I2C HAL驱动存在已知的BUSY标志Bug。当I2C总线因噪声  │
  │ 导致BUSY标志被错误置位后,所有后续I2C操作返回HAL_BUSY/HAL_ERROR,  │
  │ MPU6050_GetAngle()持续返回失败,导致舵机目标位置永远不更新。       │
  │                                                                     │
  │ 原代码在每次I2C错误时都调用I2C_BusRecovery(),该操作会完全重置I2C  │
  │ 外设并产生9个SCL时钟脉冲,可能干扰PCA9685的PWM输出,造成舵机抖动。  │
  └─────────────────────────────────────────────────────────────────────┘

  问题2: 底座舵机无规律转动
  ┌─────────────────────────────────────────────────────────────────────┐
  │ 根因: 两个因素叠加:                                                │
  │ 1. ADC输入浮空(电位器未连接):浮空ADC引脚读取值随机波动0~4095,    │
  │    如果意外切换到手动模式,底座角度 = ADC*180/4095 会随机变化      │
  │ 2. I2C总线恢复时产生的时钟脉冲可能被PCA9685误解析为有效数据       │
  └─────────────────────────────────────────────────────────────────────┘

  问题3: MPU6050移动时舵机不响应
  ┌─────────────────────────────────────────────────────────────────────┐
  │ 根因: I2C BUSY标志Bug导致MPU6050通信持续失败,角度数据无法读取。  │
  │ 即使通信偶尔成功,互补滤波器在初始化后角度从0开始,需要时间收敛,   │
  │ 而原代码的dt验证逻辑在dt>50ms时使用简单平均而非互补滤波,导致     │
  │ 首次调用时角度计算不准确。                                         │
  └─────────────────────────────────────────────────────────────────────┘

13.3 修复方案

  修复1: I2C BUSY标志自动清除 (i2c.c)
  ┌─────────────────────────────────────────────────────────────────────┐
  │ 新增I2C_ClearBusyFlag()函数:                                       │
  │ - 检查I2C1->SR2的BUSY标志位(寄存器直接访问,不依赖HAL宏)          │
  │ - 如果SCL和SDA都为高但BUSY标志置位(噪声导致的假BUSY),            │
  │   通过禁用/重新使能I2C外设来清除标志                               │
  │ - 新增I2C_IsBusReady()函数,在每次I2C操作前检查并清除BUSY标志     │
  │ - 新增I2C_SafeMemWrite/I2C_SafeMemRead安全封装函数               │
  │ - I2C超时从100ms降低到50ms,减少阻塞时间                          │
  │ - 总线恢复后增加2ms延时,确保I2C外设稳定                           │
  └─────────────────────────────────────────────────────────────────────┘

  修复2: MPU6050错误处理优化 (mpu6050.c)
  ┌─────────────────────────────────────────────────────────────────────┐
  │ - 使用I2C_SafeMemRead/I2C_SafeMemWrite替代直接HAL调用             │
  │ - 新增连续错误计数器,仅在连续5次错误后才触发总线恢复              │
  │ - 新增数据有效性验证:检查加速度计幅值(0.5g~2.5g)和全零检测      │
  │ - 初始化时先执行软件复位(0x80),等待100ms后再配置                 │
  │ - 禁用中断使能(INT_ENABLE=0x00)和FIFO(USER_CTRL=0x00),          │
  │   避免中断和FIFO对轮询读取的干扰                                  │
  │ - 初始化后增加10次读取测试,确认数据通路正常                       │
  │ - 初始化完成后从加速度计计算初始角度,避免从0°开始收敛            │
  │ - 校准前增加200ms稳定等待,校准采样数从100增加到200               │
  │ - 互补滤波alpha从0.98调整为0.96,增加加速度计权重,减少漂移       │
  │ - dt有效范围从0.001~0.05s调整为0.001~0.1s,适应更长的采样间隔    │
  │ - 新增MPU6050_PrintDebugInfo()周期性诊断输出                      │
  └─────────────────────────────────────────────────────────────────────┘

  修复3: 底座舵机速率限制 (servo.c)
  ┌─────────────────────────────────────────────────────────────────────┐
  │ - 底座舵机最大步进从2°/更新降低到1°/更新(BASE_MAX_STEP=1.0f)    │
  │ - 手势死区从2°增大到3°(GESTURE_DEADZONE=3.0f)                   │
  │ - 平滑步进从2°降低到1.5°(SMOOTH_STEP=1.5f)                      │
  │ - 最小移动阈值从step降低到0.5°,减少微小抖动                     │
  │ - 新增Arm_PrintDebugInfo()周期性诊断输出                          │
  └─────────────────────────────────────────────────────────────────────┘

  修复4: ADC浮空输入检测 (adc.c)
  ┌─────────────────────────────────────────────────────────────────────┐
  │ - 新增ADC_IsKnobConnected()函数,返回旋钮连接状态                 │
  │ - 新增ADC_DetectKnobConnection()函数:                             │
  │   · 记录最近8次ADC读数                                            │
  │   · 计算最大值与最小值之差(范围)                                  │
  │   · 范围<100且稳定50次以上 → 判定为已连接                         │
  │   · 范围>2000 → 判定为浮空(未连接)                               │
  │ - 浮空时ADC_ReadKnob()返回2048(对应90°中位)                      │
  │ - IIR低通滤波: filtered = (filtered*7 + raw) / 8                 │
  │ - 突变抑制: 相邻两次差值>100时取平均                              │
  │ - ADC采样时间从71.5周期保持不变(足够精度)                        │
  │ - ADC超时从10ms增加到50ms,减少超时错误                           │
  └─────────────────────────────────────────────────────────────────────┘

  修复5: 主循环诊断输出 (main.c)
  ┌─────────────────────────────────────────────────────────────────────┐
  │ - 新增每秒1次周期性诊断输出(DIAG_PRINT_INTERVAL_MS=1000)         │
  │ - 输出MPU6050角度、错误计数、数据有效性                           │
  │ - 输出舵机目标/当前位置和当前模式                                 │
  │ - 输出I2C错误/恢复计数和旋钮连接状态                             │
  │ - MPU6050连续失败10次时打印警告                                   │
  │ - MPU6050读取间隔从10ms调整为20ms,与主循环同步                   │
  │ - 手动模式下调用Arm_DetectKnob()检测旋钮状态                     │
  └─────────────────────────────────────────────────────────────────────┘
