## 1. MPU6050概述与技术规格

### 1.1 芯片简介

MPU6050是由InvenSense（现被TDK收购）设计的6轴运动追踪芯片，集成**3轴MEMS陀螺仪**和**3轴MEMS加速度计**于同一硅片上。芯片内置**数字运动处理器（DMP, Digital Motion Processor）**，可 offload 主控制器的姿态计算负担。MPU6050采用4×4×0.9mm QFN封装，通过I2C接口通信，广泛应用于无人机、机器人、可穿戴设备和手势控制系统。

**核心优势**：

- **单芯片6轴方案**：加速度计和陀螺仪共用同一MEMS基板，消除轴间对准误差
- **内置DMP**：硬件级传感器融合，直接输出四元数或欧拉角，降低主控负载
- **高精度ADC**：每通道独立的16位模数转换器
- **可编程量程**：多种满量程范围可选，适配不同应用场景
- **辅助I2C接口**：可外接磁力计（如HMC5883L）扩展为9轴系统

### 1.2 关键技术参数

| 参数项 | 陀螺仪（Gyroscope） | 加速度计（Accelerometer） |
|---|---|---|
| **满量程范围** | ±250, ±500, ±1000, ±2000 °/s | ±2g, ±4g, ±8g, ±16g |
| **ADC分辨率** | 16位（有符号） | 16位（有符号） |
| **灵敏度因子（LSB/单位）** | 131 / 65.5 / 32.8 / 16.4 LSB/(°/s) | 16384 / 8192 / 4096 / 2048 LSB/g |
| **正常工作电流** | 3.6 mA | 500 μA |
| **休眠模式电流** | 5 μA | 5 μA |
| **输出数据率** | 最高8kHz（DMP模式）/ 1kHz（寄存器模式） | 最高1kHz |
| **内置低通滤波器** | 可编程：260/184/94/44/21/10/5 Hz | 可编程 |
| **零偏校准公差** | ±20 °/s（X/Y轴） | ±50 mg（X/Y轴） |
| **温度传感器** | -40°C ~ +85°C，灵敏度340 LSB/°C | — |

**供电与接口参数**：

| 参数项 | 规格值 |
|---|---|
| 工作电压 | 2.375V ~ 3.46V（芯片）；3.3V ~ 5V（GY-521模块） |
| 通信接口 | I2C（最高400kHz快速模式）；辅助I2C接口 |
| I2C地址 | 0x68（AD0引脚接地）；0x69（AD0引脚接VCC） |
| 芯片ID（WHO_AM_I） | 0x68（只读） |
| 工作温度 | -40°C ~ +85°C |

> **重要提示**：芯片的3.3V逻辑电平与Arduino Uno等5V单片机连接时，GY-521模块已内置电平转换和3.3V稳压器，可直接使用5V供电。若直接使用裸芯片，必须搭配电平转换电路。

## 2. 硬件模块与引脚定义

### 2.1 GY-521模块引脚图

MPU6050最常见的开发模块为GY-521，集成稳压器、上拉电阻和排针接口，可直接连接Arduino等开发板。

```
GY-521 MPU6050 Module (Top View)

   模块尺寸：20mm × 15mm × 2mm
```

### 2.2 引脚功能说明

| 引脚名称 | 类型 | 描述 |
|---|---|---|
| **VCC** | 电源输入 | 供电引脚，接3.3V或5V（模块内置稳压器） |
| **GND** | 电源地 | 接地引脚 |
| **SCL** | 数字I/O | I2C时钟线，连接MCU的SCL引脚 |
| **SDA** | 数字I/O | I2C数据线，连接MCU的SDA引脚 |
| **XCL** | 数字I/O | 辅助I2C时钟线，用于连接外部传感器（如磁力计） |
| **XDA** | 数字I/O | 辅助I2C数据线，用于连接外部传感器 |
| **AD0** | 数字输入 | I2C地址选择引脚。接GND→地址0x68；接VCC→地址0x69 |
| **INT** | 数字输出 | 中断输出引脚，数据就绪或运动检测时触发 |

### 2.4 多个MPU6050级联

单个I2C总线最多可挂接**2个**MPU6050模块，通过AD0引脚区分地址：

| 模块 | AD0引脚连接 | I2C地址（7位） | 读/写地址（8位） |
|---|---|---|---|
| 模块1 | GND | 0x68 | 读：0xD1 / 写：0xD0 |
| 模块2 | VCC | 0x69 | 读：0xD3 / 写：0xD2 |

如需连接更多传感器，需使用I2C多路复用器（如TCA9548A）。

## 3. I2C通信协议

### 3.1 通信基础

MPU6050通过**I2C（Inter-Integrated Circuit）**协议与主控通信，仅需两根信号线：

- **SCL（Serial Clock）**：时钟线，由主设备产生同步信号
- **SDA（Serial Data）**：数据线，双向传输

**通信时序要求**：

| 参数 | 标准模式 | 快速模式 | 说明 |
|---|---|---|---|
| SCL频率 | 100 kHz | 400 kHz | 推荐使用快速模式 |
| 起始条件（START） | SCL高电平时SDA下降沿 | 同上 | 通信开始信号 |
| 停止条件（STOP） | SCL高电平时SDA上升沿 | 同上 | 通信结束信号 |
| 数据有效性 | SCL高电平期间SDA稳定 | 同上 | 数据必须在时钟高电平期间稳定 |

### 3.2 寄存器读写时序

#### 单字节写入

```
[START] + [器件地址+W] + [ACK] + [寄存器地址] + [ACK] + [数据字节] + [ACK] + [STOP]
```

#### 单字节读取

```
[START] + [器件地址+W] + [ACK] + [寄存器地址] + [ACK]
[START] + [器件地址+R] + [ACK] + [数据字节] + [NACK] + [STOP]
```

> 注意：读取前需要先写入目标寄存器地址，然后通过重复起始条件切换为读模式。

#### 连续字节读取（Burst Read）

MPU6050支持内部地址自动递增，可连续读取多个字节：

```
[START] + [器件地址+W] + [ACK] + [起始寄存器地址] + [ACK]
[START] + [器件地址+R] + [ACK] + [数据1] + [ACK] + [数据2] + [ACK] + ... + [数据N] + [NACK] + [STOP]
```

此特性极大提高了传感器数据读取效率。例如从ACCEL_XOUT_H(0x3B)开始，可一次性读取14个字节（加速度6字节+温度2字节+陀螺仪6字节）。

### 3.3 7位与8位地址转换

| 7位地址 | 写操作（R/W=0） | 读操作（R/W=1） |
|---|---|---|
| 0x68 | 0xD0 | 0xD1 |
| 0x69 | 0xD2 | 0xD3 |

## 4. 寄存器映射详解

MPU6050共有117个寄存器，以下是手势跟随项目中最核心的寄存器。所有地址均为8位十六进制值。

### 4.1 核心寄存器速查表

| 寄存器名称 | 地址 | 读写 | 功能描述 |
|---|---|---|---|
| `SMPLRT_DIV` | 0x19 | R/W | 采样率分频器 |
| `CONFIG` | 0x1A | R/W | 低通滤波器和外部帧同步配置 |
| `GYRO_CONFIG` | 0x1B | R/W | 陀螺仪满量程范围配置 |
| `ACCEL_CONFIG` | 0x1C | R/W | 加速度计满量程范围配置 |
| `ACCEL_XOUT_H` | 0x3B | R | 加速度X轴高字节 |
| `ACCEL_XOUT_L` | 0x3C | R | 加速度X轴低字节 |
| `ACCEL_YOUT_H` | 0x3D | R | 加速度Y轴高字节 |
| `ACCEL_YOUT_L` | 0x3E | R | 加速度Y轴低字节 |
| `ACCEL_ZOUT_H` | 0x3F | R | 加速度Z轴高字节 |
| `ACCEL_ZOUT_L` | 0x40 | R | 加速度Z轴低字节 |
| `TEMP_OUT_H` | 0x41 | R | 温度传感器高字节 |
| `TEMP_OUT_L` | 0x42 | R | 温度传感器低字节 |
| `GYRO_XOUT_H` | 0x43 | R | 陀螺仪X轴高字节 |
| `GYRO_XOUT_L` | 0x44 | R | 陀螺仪X轴低字节 |
| `GYRO_YOUT_H` | 0x45 | R | 陀螺仪Y轴高字节 |
| `GYRO_YOUT_L` | 0x46 | R | 陀螺仪Y轴低字节 |
| `GYRO_ZOUT_H` | 0x47 | R | 陀螺仪Z轴高字节 |
| `GYRO_ZOUT_L` | 0x48 | R | 陀螺仪Z轴低字节 |
| `PWR_MGMT_1` | 0x6B | R/W | 电源管理1：时钟源、休眠、复位 |
| `PWR_MGMT_2` | 0x6C | R/W | 电源管理2：各轴待机控制 |
| `WHO_AM_I` | 0x75 | R | 器件ID（只读，固定为0x68） |

### 4.2 关键寄存器位定义

#### PWR_MGMT_1 (0x6B) — 电源管理

| 位 | 名称 | 功能 |
|---|---|---|
| D7 | DEVICE_RESET | 写1复位所有寄存器为默认值，复位完成后自动清0 |
| D6 | SLEEP | 写1进入睡眠模式（低功耗） |
| D5 | CYCLE | 写1使能唤醒循环（与休眠配合使用） |
| D4 | — | 保留 |
| D3 | TEMP_DIS | 写1禁用温度传感器 |
| D2:D0 | CLKSEL[2:0] | 时钟源选择：0=内部8MHz；1=X轴陀螺仪PLL；2=Y轴；3=Z轴；4=外部32.768kHz；5=外部19.2MHz |

**推荐初始化值**：`0x01` — 使用X轴陀螺仪PLL作为时钟源（最稳定），不复位，不睡眠。

> **注意**：模块上电后默认进入睡眠模式（SLEEP=1），必须向PWR_MGMT_1写入0x01唤醒设备才能正常读取数据。

#### GYRO_CONFIG (0x1B) — 陀螺仪配置

| 位 | 名称 | 功能 |
|---|---|---|
| D7:D6 | XG_ST / YG_ST / ZG_ST | 自测使能位（分别对应X/Y/Z轴） |
| D4:D3 | FS_SEL[1:0] | 满量程选择：00=±250°/s；01=±500°/s；10=±1000°/s；11=±2000°/s |
| D2:D0 | — | 保留 |

**常用配置值**：

| FS_SEL值 | 写入值（左移3位） | 满量程 | 灵敏度(LSB/(°/s)) |
|---|---|---|---|
| 00 | 0x00 | ±250 °/s | 131 |
| 01 | 0x08 | ±500 °/s | 65.5 |
| 10 | 0x10 | ±1000 °/s | 32.8 |
| 11 | 0x18 | ±2000 °/s | 16.4 |

#### ACCEL_CONFIG (0x1C) — 加速度计配置

| 位 | 名称 | 功能 |
|---|---|---|
| D7:D5 | XA_ST / YA_ST / ZA_ST | 自测使能位 |
| D4:D3 | AFS_SEL[1:0] | 满量程选择：00=±2g；01=±4g；10=±8g；11=±16g |
| D2:D0 | — | 保留 |

**常用配置值**：

| AFS_SEL值 | 写入值（左移3位） | 满量程 | 灵敏度(LSB/g) |
|---|---|---|---|
| 00 | 0x00 | ±2g | 16384 |
| 01 | 0x08 | ±4g | 8192 |
| 10 | 0x10 | ±8g | 4096 |
| 11 | 0x18 | ±16g | 2048 |

#### CONFIG (0x1A) — 配置寄存器

| 位 | 名称 | 功能 |
|---|---|---|
| D5:D3 | EXT_SYNC_SET | 外部帧同步配置 |
| D2:D0 | DLPF_CFG | 数字低通滤波器配置 |

**DLPF_CFG常用配置**：

| DLPF_CFG | 加速度带宽 | 陀螺仪带宽 | 延迟 | 说明 |
|---|---|---|---|---|
| 0 | 260 Hz | 256 Hz | 0.98ms | 最低延迟，最少滤波 |
| 1 | 184 Hz | 188 Hz | 1.9ms | 轻量滤波 |
| 2 | 94 Hz | 98 Hz | 2.8ms | **推荐值，平衡性能与噪声** |
| 3 | 44 Hz | 42 Hz | 4.8ms | 较强滤波 |
| 4 | 21 Hz | 20 Hz | 8.3ms | 强滤波 |
| 5 | 10 Hz | 10 Hz | 13.4ms | 强滤波 |
| 6 | 5 Hz | 5 Hz | 18.6ms | 最强滤波 |

#### SMPLRT_DIV (0x19) — 采样率分频器

采样率计算公式：

```
当DLPF_CFG=0时：采样率 = 8kHz / (1 + SMPLRT_DIV)
当DLPF_CFG≠0时：采样率 = 1kHz / (1 + SMPLRT_DIV)
```

- `SMPLRT_DIV = 0x00`：采样率 = 1kHz（最大）
- `SMPLRT_DIV = 0x04`：采样率 = 200Hz
- `SMPLRT_DIV = 0x09`：采样率 = 100Hz

## 5. 传感器初始化和配置

### 5.1 完整初始化流程

正确的初始化是MPU6050稳定工作的前提。以下是推荐的标准初始化序列：

```c
#include "Wire.h"
#include "MPU6050_Reg.h"

#define MPU6050_ADDRESS  MPU6050_ADDR  // 0x68

/* ========== I2C底层操作函数 ========== */

// 向指定寄存器写入1字节
void MPU6050_WriteReg(uint8_t reg, uint8_t data) {
    Wire.beginTransmission(MPU6050_ADDRESS);
    Wire.write(reg);
    Wire.write(data);
    Wire.endTransmission(true);
}

// 从指定寄存器读取1字节
uint8_t MPU6050_ReadReg(uint8_t reg) {
    Wire.beginTransmission(MPU6050_ADDRESS);
    Wire.write(reg);
    Wire.endTransmission(false);  // 重复起始条件
    Wire.requestFrom(MPU6050_ADDRESS, 1);
    return Wire.available() ? Wire.read() : 0;
}

// 从指定寄存器连续读取多字节
void MPU6050_ReadRegs(uint8_t reg, uint8_t *buf, uint8_t len) {
    Wire.beginTransmission(MPU6050_ADDRESS);
    Wire.write(reg);
    Wire.endTransmission(false);
    Wire.requestFrom(MPU6050_ADDRESS, len);
    for (uint8_t i = 0; i < len && Wire.available(); i++) {
        buf[i] = Wire.read();
    }
}

/* ========== 初始化函数 ========== */

uint8_t MPU6050_Init(void) {
    uint8_t id;
    
    Wire.begin();           // 初始化I2C总线
    Wire.setClock(400000);  // 设置为400kHz快速模式
    
    // 步骤1：验证设备ID
    id = MPU6050_ReadReg(MPU6050_WHO_AM_I);
    if (id != 0x68) {
        return 1;  // 设备未找到或通信失败
    }
    
    // 步骤2：复位MPU6050
    MPU6050_WriteReg(MPU6050_PWR_MGMT_1, 0x80);
    delay(100);  // 等待复位完成
    
    // 步骤3：配置时钟源（X轴陀螺仪PLL）并退出睡眠
    MPU6050_WriteReg(MPU6050_PWR_MGMT_1, 0x01);
    
    // 步骤4：配置数字低通滤波器（DLPF=2，94Hz带宽）
    MPU6050_WriteReg(MPU6050_CONFIG, 0x02);
    
    // 步骤5：配置采样率分频器（1kHz/(1+4)=200Hz）
    MPU6050_WriteReg(MPU6050_SMPLRT_DIV, 0x04);
    
    // 步骤6：配置陀螺仪量程（±2000°/s，16.4 LSB/(°/s)）
    MPU6050_WriteReg(MPU6050_GYRO_CONFIG, 0x18);
    
    // 步骤7：配置加速度计量程（±2g，16384 LSB/g）
    MPU6050_WriteReg(MPU6050_ACCEL_CONFIG, 0x00);
    
    // 步骤8：关闭中断（简化配置）
    MPU6050_WriteReg(0x38, 0x00);  // INT_ENABLE
    
    // 步骤9：退出待机模式
    MPU6050_WriteReg(MPU6050_PWR_MGMT_2, 0x00);
    
    return 0;  // 初始化成功
}
```

### 5.2 使用Adafruit库简化初始化

对于Arduino平台，推荐使用成熟的库简化开发：

```cpp
#include <Adafruit_MPU6050.h>
#include <Adafruit_Sensor.h>
#include <Wire.h>

Adafruit_MPU6050 mpu;

void setup() {
```

## 6. 原始数据读取与转换

### 6.1 数据寄存器结构

MPU6050的数据寄存器以高字节在前（Big-Endian）的方式存储16位有符号整数。所有传感器数据（加速度、温度、陀螺仪）连续排列在地址0x3B~0x48：

| 寄存器地址 | 内容 | 字节序 |
|---|---|---|
| 0x3B ~ 0x3C | Accel_X | 高字节 + 低字节 |
| 0x3D ~ 0x3E | Accel_Y | 高字节 + 低字节 |
| 0x3F ~ 0x40 | Accel_Z | 高字节 + 低字节 |
| 0x41 ~ 0x42 | Temperature | 高字节 + 低字节 |
| 0x43 ~ 0x44 | Gyro_X | 高字节 + 低字节 |
| 0x45 ~ 0x46 | Gyro_Y | 高字节 + 低字节 |
| 0x47 ~ 0x48 | Gyro_Z | 高字节 + 低字节 |

### 6.2 原始数据读取函数

```c
/* ========== 数据结构 ========== */
typedef struct {
    int16_t accelX;
    int16_t accelY;
    int16_t accelZ;
    int16_t gyroX;
    int16_t gyroY;
    int16_t gyroZ;
    float temperature;
} MPU6050_Data_t;

/* ========== 数据读取函数 ========== */
void MPU6050_ReadData(MPU6050_Data_t *data) {
    uint8_t buf[14];  // 14字节缓冲区
    
    // 从ACCEL_XOUT_H(0x3B)开始连续读取14字节
    MPU6050_ReadRegs(MPU6050_ACCEL_XOUT_H, buf, 14);
    
    // 合并高字节和低字节（有符号16位，大端序）
    data->accelX = (int16_t)((buf[0] << 8) | buf[1]);
    data->accelY = (int16_t)((buf[2] << 8) | buf[3]);
    data->accelZ = (int16_t)((buf[4] << 8) | buf[5]);
    
    // 温度数据
    int16_t tempRaw = (int16_t)((buf[6] << 8) | buf[7]);
    data->temperature = (tempRaw / 340.0) + 36.53;  // 公式来自数据手册
    
    // 陀螺仪数据
    data->gyroX = (int16_t)((buf[8] << 8) | buf[9]);
    data->gyroY = (int16_t)((buf[10] << 8) | buf[11]);
    data->gyroZ = (int16_t)((buf[12] << 8) | buf[13]);
}
```

#### 加速度计转换

原始值需除以对应满量程的灵敏度因子，得到以"g"为单位的加速度值（1g = 9.80665 m/s²）：

```c

float gy_dps = data.gyroY / GYRO_SCALE_FACTOR;
```

## 7. 传感器校准

### 7.1 为什么需要校准

MPU6050出厂时存在制造偏差，未经校准的传感器在静止状态下也会有显著的非零读数。主要误差来源包括：

- **零点偏移（Offset/Bias）**：传感器静止时的非零输出
- **比例因子误差**：灵敏度与标称值的偏差
- **轴间耦合**：正交轴之间的串扰
- **温度漂移**：随温度变化的参数偏移
- **噪声**：随机测量误差

**校准目标**：在传感器静止平放状态下，确保：

- 加速度计X、Y轴读数 ≈ 0，Z轴读数 ≈ 1g（对应原始值16384@±2g）
- 陀螺仪三轴读数 ≈ 0

### 7.2 陀螺仪零点校准

陀螺仪零点校准是最关键的步骤。传感器静止时，采集大量样本取平均值作为偏移量，之后每次读数减去该偏移。

```c
/* ========== 陀螺仪校准 ========== */
typedef struct {
    int16_t gyroOffsetX;
    int16_t gyroOffsetY;
    int16_t gyroOffsetZ;
    int16_t accelOffsetX;
    int16_t accelOffsetY;
    int16_t accelOffsetZ;
} MPU6050_Offset_t;

MPU6050_Offset_t offset;

void MPU6050_CalibrateGyro(MPU6050_Offset_t *off, uint16_t samples) {
    int32_t sumX = 0, sumY = 0, sumZ = 0;
    MPU6050_Data_t data;
    
    Serial.println("陀螺仪校准中... 请保持传感器完全静止！");
    delay(2000);
    
    // 采集样本
    for (uint16_t i = 0; i < samples; i++) {
        MPU6050_ReadData(&data);
        sumX += data.gyroX;
        sumY += data.gyroY;
        sumZ += data.gyroZ;
        delay(2);  // 每2ms采样一次
    }
    
    off->gyroOffsetX = (int16_t)(sumX / samples);
    off->gyroOffsetY = (int16_t)(sumY / samples);
    off->gyroOffsetZ = (int16_t)(sumZ / samples);
    
    Serial.println("陀螺仪校准完成！");
    Serial.print("X轴偏移: "); Serial.println(off->gyroOffsetX);
    Serial.print("Y轴偏移: "); Serial.println(off->gyroOffsetY);
    Serial.print("Z轴偏移: "); Serial.println(off->gyroOffsetZ);
}
```

**校准建议**：

- 采样数量：至少1000次（约2秒），2000次更佳
- 环境要求：传感器完全静止，无振动
- 预热时间：上电后等待2-3分钟再进行校准，减少温度漂移影响
- 存储偏移值：将校准后的偏移写入Arduino EEPROM或代码常量中，避免每次上电重新校准

### 7.3 加速度计校准

加速度计校准较复杂，通常需要6面校准（正X、负X、正Y、负Y、正Z、负Z方向各朝上静止放置）。对于机械臂手势跟随项目，简化的Z轴校准即可满足需求：

```c
void MPU6050_CalibrateAccel(MPU6050_Offset_t *off, uint16_t samples) {
    int32_t sumX = 0, sumY = 0, sumZ = 0;
    MPU6050_Data_t data;
    
    Serial.println("加速度计校准中... 请保持传感器平放静止！");
    delay(2000);
    
    for (uint16_t i = 0; i < samples; i++) {
        MPU6050_ReadData(&data);
        sumX += data.accelX;
        sumY += data.accelY;
        sumZ += (data.accelZ - 16384);  // 减去1g重力分量
        delay(2);
    }
    
    off->accelOffsetX = (int16_t)(sumX / samples);
    off->accelOffsetY = (int16_t)(sumY / samples);
    off->accelOffsetZ = (int16_t)(sumZ / samples);
    
    Serial.println("加速度计校准完成！");
}
```

## 8. 姿态解算算法

### 8.1 传感器特性对比

| 特性 | 加速度计 | 陀螺仪 |
|---|---|---|
| **静态精度** | 高（可通过重力向量测倾角） | 低（存在零点偏移） |
| **动态精度** | 低（受运动加速度干扰） | 高（测量角速度） |
| **长期稳定性** | 好（不受漂移影响） | 差（积分累积误差） |
| **短期稳定性** | 差（噪声大） | 好（噪声小） |
| **Yaw测量** | 无法测量 | 可积分但会漂移 |

**核心问题**：加速度计长期稳定但短期噪声大；陀螺仪短期精确但长期积分漂移。姿态解算的本质是**融合两者数据**，各取所长。

### 8.2 互补滤波（Complementary Filter）

互补滤波是最简单高效的姿态融合方法，适合资源受限的嵌入式平台。

#### 原理

互补滤波将陀螺仪数据通过**高通滤波器**（信任短期变化），将加速度计数据通过**低通滤波器**（信任长期趋势），两者加权融合：

```
angle = alpha * (angle + gyro_rate * dt) + (1 - alpha) * accel_angle
```

其中 `alpha` 通常取 **0.96 ~ 0.98**，表示更信任陀螺仪的短期精度。

#### Roll和Pitch角度计算

```c
#include <math.h>

#define RAD_TO_DEG  57.2957795f   // 180/PI
#define ALPHA       0.96f         // 互补滤波系数

float roll = 0.0f;   // 横滚角（绕X轴）
pitch = 0.0f;  // 俯仰角（绕Y轴）

void ComplementaryFilter(int16_t ax, int16_t ay, int16_t az,
                         int16_t gx, int16_t gy, int16_t gz,
                         float dt, float *roll_out, float *pitch_out) {
    // 1. 从加速度计计算倾角（仅适用于静止或匀速运动状态）
    // 注意：sqrt(ay² + az²) 是为了归一化
    float accelRoll  = atan2f((float)ay, sqrtf((float)ax*ax + (float)az*az)) * RAD_TO_DEG;
    float accelPitch = atan2f(-(float)ax, sqrtf((float)ay*ay + (float)az*az)) * RAD_TO_DEG;
    
    // 2. 陀螺仪角速度（°/s），已减去偏移
    float gyroRollRate  = (float)gx / 131.0f;   // 假设±250°/s量程
    float gyroPitchRate = (float)gy / 131.0f;
    
    // 3. 互补滤波融合
    roll  = ALPHA * (roll  + gyroRollRate  * dt) + (1.0f - ALPHA) * accelRoll;
    pitch = ALPHA * (pitch + gyroPitchRate * dt) + (1.0f - ALPHA) * accelPitch;
    
    *roll_out = roll;
    *pitch_out = pitch;
}
```

#### 完整互补滤波示例（Arduino）

```cpp
#include <Wire.h>
#include <math.h>

// 引脚和地址定义
#define MPU6050_ADDR  0x68
#define RAD_TO_DEG    57.295779513082320876798154814105f

// 互补滤波系数（0.0~1.0，越大越信任陀螺仪）
#define COMP_FILTER_ALPHA  0.96f

// 校准偏移（需预先校准获取）
int16_t gyroOffsetX = 0, gyroOffsetY = 0, gyroOffsetZ = 0;

// 角度变量
float roll = 0, pitch = 0, yaw = 0;
unsigned long lastTime = 0;

void setup() {
    Serial.begin(115200);
    Wire.begin();
    Wire.setClock(400000);
    
    // 初始化MPU6050
    Wire.beginTransmission(MPU6050_ADDR);
    Wire.write(0x6B);  // PWR_MGMT_1
    Wire.write(0x01);  // 时钟源=X轴陀螺仪PLL
    Wire.endTransmission();
    
    Wire.beginTransmission(MPU6050_ADDR);
    Wire.write(0x1B);  // GYRO_CONFIG
    Wire.write(0x00);  // ±250°/s
    Wire.endTransmission();
    
    Wire.beginTransmission(MPU6050_ADDR);
    Wire.write(0x1C);  // ACCEL_CONFIG
    Wire.write(0x00);  // ±2g
    Wire.endTransmission();
    
    // 校准陀螺仪
    calibrateGyro();
    
    lastTime = micros();
}

void calibrateGyro() {
    long sumX = 0, sumY = 0, sumZ = 0;
    const int samples = 2000;
    
    Serial.println("校准陀螺仪...保持静止");
    delay(2000);
    
    for (int i = 0; i < samples; i++) {
        int16_t gx, gy, gz;
        readGyroRaw(&gx, &gy, &gz);
        sumX += gx; sumY += gy; sumZ += gz;
        delay(1);
    }
    
    gyroOffsetX = sumX / samples;
    gyroOffsetY = sumY / samples;
    gyroOffsetZ = sumZ / samples;
    
    Serial.println("校准完成！");
}

void readAccelRaw(int16_t *ax, int16_t *ay, int16_t *az) {
    Wire.beginTransmission(MPU6050_ADDR);
    Wire.write(0x3B);
    Wire.endTransmission(false);
    Wire.requestFrom(MPU6050_ADDR, 6);
    *ax = (Wire.read() << 8) | Wire.read();
    *ay = (Wire.read() << 8) | Wire.read();
    *az = (Wire.read() << 8) | Wire.read();
}

void readGyroRaw(int16_t *gx, int16_t *gy, int16_t *gz) {
    Wire.beginTransmission(MPU6050_ADDR);
    Wire.write(0x43);
    Wire.endTransmission(false);
    Wire.requestFrom(MPU6050_ADDR, 6);
    *gx = (Wire.read() << 8) | Wire.read();
    *gy = (Wire.read() << 8) | Wire.read();
    *gz = (Wire.read() << 8) | Wire.read();
}

void loop() {
    int16_t ax, ay, az, gx, gy, gz;
    
    readAccelRaw(&ax, &ay, &az);
    readGyroRaw(&gx, &gy, &gz);
    
    // 减去偏移
    gx -= gyroOffsetX;
    gy -= gyroOffsetY;
    gz -= gyroOffsetZ;
    
    // 计算时间间隔
    unsigned long currentTime = micros();
    float dt = (currentTime - lastTime) / 1000000.0f;
    lastTime = currentTime;
    
    // 加速度计计算角度
    float accRoll  = atan2f((float)ay, sqrtf((float)ax*ax + (float)az*az)) * RAD_TO_DEG;
    float accPitch = atan2f(-(float)ax, sqrtf((float)ay*ay + (float)az*az)) * RAD_TO_DEG;
    
    // 陀螺仪角速度转°/s
    float gyroX_dps = gx / 131.0f;
    float gyroY_dps = gy / 131.0f;
    float gyroZ_dps = gz / 131.0f;
    
    // 互补滤波
    roll  = COMP_FILTER_ALPHA * (roll  + gyroX_dps * dt) + (1.0f - COMP_FILTER_ALPHA) * accRoll;
    pitch = COMP_FILTER_ALPHA * (pitch + gyroY_dps * dt) + (1.0f - COMP_FILTER_ALPHA) * accPitch;
    yaw  += gyroZ_dps * dt;  // Yaw仅由陀螺仪积分，会漂移
    
    // 输出结果
    Serial.print("Roll: "); Serial.print(roll, 1);
    Serial.print(" | Pitch: "); Serial.print(pitch, 1);
    Serial.print(" | Yaw: "); Serial.println(yaw, 1);
    
    delay(10);  // 约100Hz刷新率
}
```

### 8.3 DMP（数字运动处理器）方法

DMP是MPU6050内部的专用协处理器，直接输出融合后的四元数或欧拉角，无需主控进行复杂计算。

#### DMP优势

| 特性 | 互补滤波（软件） | DMP（硬件） |
|---|---|---|
| CPU负载 | 高（浮点运算） | 极低（直接读取结果） |
| 精度 | 取决于参数调优 | InvenSense官方优化算法 |
| Yaw漂移 | 严重（无磁力计） | 较轻（内置补偿） |
| 开发难度 | 中等 | 较高（需加载固件） |
| 输出格式 | 需自行计算 | 四元数/欧拉角直接输出 |

#### DMP初始化（使用I2Cdevlib库）

```cpp
#include "I2Cdev.h"
#include "MPU6050_6Axis_MotionApps20.h"

MPU6050 mpu;

// DMP状态变量
uint8_t devStatus;
uint16_t packetSize;
uint8_t fifoBuffer[64];

// 姿态数据
Quaternion q;
VectorFloat gravity;
float ypr[3];  // yaw, pitch, roll（弧度）

void setupDMP() {
    Wire.begin();
    Wire.setClock(400000);
    
    mpu.initialize();
    
    // 加载DMP固件
    devStatus = mpu.dmpInitialize();
    
    // 设置偏移（需预先校准）
    mpu.setXGyroOffset(220);
    mpu.setYGyroOffset(76);
    mpu.setZGyroOffset(-85);
    mpu.setZAccelOffset(1688);
    
    if (devStatus == 0) {
        mpu.CalibrateAccel(6);
        mpu.CalibrateGyro(6);
        mpu.setDMPEnabled(true);  // 启用DMP
        packetSize = mpu.dmpGetFIFOPacketSize();
    }
}

void readDMPAngles(float *yaw, float *pitch, float *roll) {
    if (mpu.dmpGetCurrentFIFOPacket(fifoBuffer)) {
        // 解析四元数
        mpu.dmpGetQuaternion(&q, fifoBuffer);
        // 解析重力向量
        mpu.dmpGetGravity(&gravity, &q);
        // 计算欧拉角（弧度）
        mpu.dmpGetYawPitchRoll(ypr, &q, &gravity);
        
        *yaw   = ypr[0] * 180.0f / M_PI;   // 转角度
        *pitch = ypr[1] * 180.0f / M_PI;
        *roll  = ypr[2] * 180.0f / M_PI;
    }
}
```

#### 四元数转欧拉角公式

如需手动将DMP输出的四元数转换为欧拉角：

```c
// 四元数结构体
typedef struct {
    float w, x, y, z;
} Quaternion_t;

// 四元数转欧拉角（Z-Y-X旋转顺序）
void QuaternionToEuler(Quaternion_t *q, float *roll, float *pitch, float *yaw) {
    *roll  = atan2f(2.0f * (q->w * q->x + q->y * q->z), 
                     1.0f - 2.0f * (q->x * q->x + q->y * q->y)) * RAD_TO_DEG;
    
    *pitch = asinf(2.0f * (q->w * q->y - q->z * q->x)) * RAD_TO_DEG;
    
    *yaw   = atan2f(2.0f * (q->w * q->z + q->x * q->y), 
                     1.0f - 2.0f * (q->y * q->y + q->z * q->z)) * RAD_TO_DEG;
}
```

> **重要提示**：DMP初始化时模块必须**水平静止放置**，否则自检会失败返回错误码8。调用`dmpInitialize()`前确保MPU6050平放且无振动。

### 8.4 姿态解算算法选择建议

| 应用场景 | 推荐算法 | 理由 |
|---|---|---|
| Arduino UNO（资源有限） | 互补滤波 | 简单高效，无需额外库 |
| ESP32/STM32（资源充裕） | DMP 或 互补滤波 | DMP精度高；互补滤波更灵活 |
| 需要最小延迟 | DMP | 内置优化，延迟最低 |
| 需要完全控制算法 | 互补滤波/卡尔曼 | 可自定义参数和融合逻辑 |
| 仅控制2轴（Roll+Pitch） | 互补滤波 | 足够满足需求 |
| 需要Yaw角（3轴全控） | DMP | Yaw漂移更小 |

## 9. 舵机控制与机械臂映射

### 9.1 舵机基础

舵机（Servo）是最常用的执行器之一，通过PWM信号控制角度位置。常见规格：

| 参数 | 典型值 |
|---|---|
| 控制信号 | PWM，周期20ms（50Hz） |
| 脉宽范围 | 500μs ~ 2500μs |
| 角度范围 | 0° ~ 180°（部分支持270°） |
| 供电电压 | 4.8V ~ 6.0V |
| 供电电流 | 100mA（静态）/ 500mA+（堵转） |

**Arduino Servo库使用**：

```cpp
#include <Servo.h>

Servo servo1;  // 创建舵机对象

void setup() {
    servo1.attach(9);        // 舵机信号线接数字引脚9
    servo1.write(90);        // 设置初始角度为90度（中位）
}

void loop() {
    servo1.write(45);        // 转到45度
    delay(1000);
    servo1.write(135);       // 转到135度
    delay(1000);
}
```

### 9.2 姿态角度到舵机角度的映射

MPU6050输出的Roll、Pitch、Yaw角度范围通常为 **-90° ~ +90°**（或-180°~+180°），需要映射到舵机的 **0° ~ 180°** 范围。

```cpp
/* ========== 角度映射函数 ========== */

// 将传感器角度映射到舵机角度
float mapAngle(float sensorAngle, float sensorMin, float sensorMax, 
               float servoMin, float servoMax) {
    // 使用Arduino内置map的浮点版本
    return servoMin + (sensorAngle - sensorMin) * (servoMax - servoMin) 
                    / (sensorMax - sensorMin);
}

// 快捷映射：±90°传感器角度 → 0~180°舵机角度
float sensorToServo(float sensorAngle) {
    return constrain(mapAngle(sensorAngle, -90.0f, 90.0f, 0.0f, 180.0f), 0.0f, 180.0f);
}

// 快捷映射（带死区）
float sensorToServoWithDeadzone(float sensorAngle, float deadzone) {
    if (fabs(sensorAngle) < deadzone) sensorAngle = 0;  // 死区内置零
    return constrain(mapAngle(sensorAngle, -90.0f, 90.0f, 0.0f, 180.0f), 0.0f, 180.0f);
}
```

### 9.3 3轴机械臂关节映射方案

3轴机械臂通常对应以下关节自由度：

| 机械臂轴 | 姿态角 | 典型功能 | 映射关系 |
|---|---|---|---|
| 基座旋转（Waist） | Yaw | 整体左右旋转 | Yaw → 舵机0~180° |
| 肩关节（Shoulder） | Pitch | 前后俯仰 | Pitch → 舵机0~180° |
| 肘关节（Elbow） | Roll | 侧向翻转 | Roll → 舵机0~180° |

**映射示例**：

```cpp
Servo servoBase;      // 基座 - Yaw控制
Servo servoShoulder;  // 肩关节 - Pitch控制
Servo servoElbow;     // 肘关节 - Roll控制

void setupServos() {
    servoBase.attach(9);
    servoShoulder.attach(10);
    servoElbow.attach(11);
    
    // 初始位置
    servoBase.write(90);
    servoShoulder.write(90);
    servoElbow.write(90);
}

void updateArmPosition(float roll, float pitch, float yaw) {
    // 映射到舵机角度
    int baseAngle     = (int)sensorToServo(yaw);
    int shoulderAngle = (int)sensorToServo(pitch);
    int elbowAngle    = (int)sensorToServo(roll);
    
    // 写入舵机
    servoBase.write(baseAngle);
    servoShoulder.write(shoulderAngle);
    servoElbow.write(elbowAngle);
}
```

### 9.4 平滑处理与限幅

直接映射会导致舵机抖动，需要加入平滑处理和限幅保护：

```cpp
// 平滑系数（0.0~1.0，越大越平滑但响应越慢）
#define SMOOTHING_FACTOR  0.15f

// 当前舵机角度（记录用于平滑）
float currentServoAngles[3] = {90.0f, 90.0f, 90.0f};

// 带平滑的角度更新
void updateServoSmooth(Servo *servo, int index, float targetAngle) {
    // 一阶低通平滑
    currentServoAngles[index] += SMOOTHING_FACTOR * (targetAngle - currentServoAngles[index]);
    
    // 限幅到安全范围
    int angle = constrain((int)currentServoAngles[index], 0, 180);
    
    servo->write(angle);
}

// 使用示例
void updateAllServos(float roll, float pitch, float yaw) {
    updateServoSmooth(&servoBase,     0, sensorToServo(yaw));
    updateServoSmooth(&servoShoulder, 1, sensorToServo(pitch));
    updateServoSmooth(&servoElbow,    2, sensorToServo(roll));
}
```

### 10.3 完整Arduino代码

```cpp
/*
 * ============================================================
 * 3轴手势跟随机械臂 — 完整项目代码
 * 基于MPU6050姿态传感器 + Arduino + 3路舵机
 * ============================================================
 * 硬件接线：
 *   MPU6050: VCC→5V, GND→GND, SCL→A5, SDA→A4
 *   舵机基座: 信号线→D9
 *   舵机肩: 信号线→D10
 *   舵机肘: 信号线→D11
 *   注意：舵机独立供电，与Arduino共地！
 * ============================================================
 */

#include <Wire.h>
#include <Servo.h>
#include <math.h>

// ====== 配置常量 ======
#define MPU6050_ADDR        0x68
#define RAD_TO_DEG          57.2957795f
#define COMP_FILTER_ALPHA   0.96f       // 互补滤波系数
#define SMOOTHING_FACTOR    0.12f       // 舵机平滑系数
#define SERVO_MIN_ANGLE     0
#define SERVO_MAX_ANGLE     180
#define SAMPLE_RATE_HZ      100         // 采样率

// ====== 舵机对象 ======
Servo servoBase;      // 基座 - 对应Yaw
Servo servoShoulder;  // 肩关节 - 对应Pitch
Servo servoElbow;     // 肘关节 - 对应Roll

// ====== 全局状态变量 ======
float roll = 0, pitch = 0, yaw = 0;
float currentServo[3] = {90.0f, 90.0f, 90.0f};  // 当前舵机角度
unsigned long lastMicros = 0;

// 陀螺仪校准偏移
int16_t gyroOffX = 0, gyroOffY = 0, gyroOffZ = 0;

// ====== I2C底层函数 ======
void mpuWrite(uint8_t reg, uint8_t data) {
    Wire.beginTransmission(MPU6050_ADDR);
    Wire.write(reg);
    Wire.write(data);
    Wire.endTransmission(true);
}

void mpuRead(uint8_t reg, uint8_t *buf, uint8_t len) {
    Wire.beginTransmission(MPU6050_ADDR);
    Wire.write(reg);
    Wire.endTransmission(false);
    Wire.requestFrom(MPU6050_ADDR, len);
    for (uint8_t i = 0; i < len && Wire.available(); i++) {
        buf[i] = Wire.read();
    }
}

// ====== MPU6050初始化 ======
void initMPU6050() {
    Wire.begin();
    Wire.setClock(400000);
    
    // 退出睡眠，设置PLL时钟
    mpuWrite(0x6B, 0x01);
    delay(100);
    
    // 配置DLPF（94Hz带宽）
    mpuWrite(0x1A, 0x02);
    
    // 配置陀螺仪量程 ±250°/s (131 LSB/(°/s))
    mpuWrite(0x1B, 0x00);
    
    // 配置加速度计 ±2g (16384 LSB/g)
    mpuWrite(0x1C, 0x00);
    
    // 配置采样率 1kHz/(1+4) = 200Hz
    mpuWrite(0x19, 0x04);
}

// ====== 陀螺仪校准 ======
void calibrateGyro() {
    int32_t sx = 0, sy = 0, sz = 0;
    const int n = 1000;
    
    Serial.println("=== 陀螺仪校准中 ===");
    Serial.println("请将MPU6050平放静止不动");
    delay(3000);
    
    for (int i = 0; i < n; i++) {
        uint8_t buf[6];
        mpuRead(0x43, buf, 6);
        sx += (int16_t)((buf[0] << 8) | buf[1]);
        sy += (int16_t)((buf[2] << 8) | buf[3]);
        sz += (int16_t)((buf[4] << 8) | buf[5]);
        delay(2);
    }
    
    gyroOffX = sx / n;
    gyroOffY = sy / n;
    gyroOffZ = sz / n;
    
    Serial.println("校准完成！");
    Serial.print("Offset X="); Serial.print(gyroOffX);
    Serial.print(" Y="); Serial.print(gyroOffY);
    Serial.print(" Z="); Serial.println(gyroOffZ);
}

// ====== 读取MPU6050全部数据 ======
void readMPU(int16_t *ax, int16_t *ay, int16_t *az,
             int16_t *gx, int16_t *gy, int16_t *gz) {
    uint8_t buf[14];
    mpuRead(0x3B, buf, 14);
    
    *ax = (int16_t)((buf[0] << 8) | buf[1]);
    *ay = (int16_t)((buf[2] << 8) | buf[3]);
    *az = (int16_t)((buf[4] << 8) | buf[5]);
    // buf[6:7] 温度，跳过
    *gx = (int16_t)((buf[8]  << 8) | buf[9])  - gyroOffX;
    *gy = (int16_t)((buf[10] << 8) | buf[11]) - gyroOffY;
    *gz = (int16_t)((buf[12] << 8) | buf[13]) - gyroOffZ;
}

// ====== 互补滤波姿态解算 ======
void updateAttitude(int16_t ax, int16_t ay, int16_t az,
                    int16_t gx, int16_t gy, int16_t gz,
                    float dt) {
    // 加速度计角度
    float accRoll  = atan2f((float)ay, sqrtf((float)ax*ax + (float)az*az)) * RAD_TO_DEG;
    float accPitch = atan2f(-(float)ax, sqrtf((float)ay*ay + (float)az*az)) * RAD_TO_DEG;
    
    // 陀螺仪角速度 (°/s)
    float gyroX = gx / 131.0f;
    float gyroY = gy / 131.0f;
    float gyroZ = gz / 131.0f;
    
    // 互补滤波
    roll  = COMP_FILTER_ALPHA * (roll  + gyroX * dt) + (1.0f - COMP_FILTER_ALPHA) * accRoll;
    pitch = COMP_FILTER_ALPHA * (pitch + gyroY * dt) + (1.0f - COMP_FILTER_ALPHA) * accPitch;
    yaw  += gyroZ * dt;  // Yaw仅积分，无磁传感器会漂移
    
    // 限制Yaw范围（防止无限累积）
    if (yaw > 180.0f) yaw -= 360.0f;
    if (yaw < -180.0f) yaw += 360.0f;
}

// ====== 角度映射 ======
float sensorToServo(float angle) {
    float servoAngle = 90.0f + angle;  // ±90° → 0~180°
    return constrain(servoAngle, SERVO_MIN_ANGLE, SERVO_MAX_ANGLE);
}

// ====== 平滑更新舵机 ======
void updateServo(int index, Servo &servo, float targetAngle) {
    currentServo[index] += SMOOTHING_FACTOR * (targetAngle - currentServo[index]);
    servo.write((int)currentServo[index]);
}

// ====== 初始化 ======
void setup() {
    Serial.begin(115200);
    Serial.println("=== 3轴手势跟随机械臂 ===");
    
    // 初始化MPU6050
    initMPU6050();
    calibrateGyro();
    
    // 初始化舵机
    servoBase.attach(9);
    servoShoulder.attach(10);
    servoElbow.attach(11);
    
    servoBase.write(90);
    servoShoulder.write(90);
    servoElbow.write(90);
    
    lastMicros = micros();
    
    Serial.println("系统启动完成！开始跟踪...");
    Serial.println("格式: Roll, Pitch, Yaw | Base, Shoulder, Elbow");
}

// ====== 主循环 ======
void loop() {
    int16_t ax, ay, az, gx, gy, gz;
    
    // 读取传感器
    readMPU(&ax, &ay, &az, &gx, &gy, &gz);
    
    // 计算时间间隔
    unsigned long now = micros();
    float dt = (now - lastMicros) / 1000000.0f;
    lastMicros = now;
    
    // 姿态解算
    updateAttitude(ax, ay, az, gx, gy, gz, dt);
    
    // 映射到舵机角度
    float targetBase     = sensorToServo(yaw);
    float targetShoulder = sensorToServo(pitch);
    float targetElbow    = sensorToServo(roll);
    
    // 平滑更新舵机
    updateServo(0, servoBase,     targetBase);
    updateServo(1, servoShoulder, targetShoulder);
    updateServo(2, servoElbow,    targetElbow);
    
    // 串口输出调试信息
    static unsigned long lastPrint = 0;
    if (millis() - lastPrint > 100) {
        lastPrint = millis();
        Serial.print("R:"); Serial.print(roll, 1);
        Serial.print(" P:"); Serial.print(pitch, 1);
        Serial.print(" Y:"); Serial.print(yaw, 1);
        Serial.print(" | S:");
        Serial.print((int)currentServo[0]); Serial.print(",");
        Serial.print((int)currentServo[1]); Serial.print(",");
        Serial.println((int)currentServo[2]);
    }
    
    // 保持固定刷新率
    delayMicroseconds(10000);  // 约100Hz
}
```

### 10.4 进阶版本：基于DMP的完整代码

如需使用DMP获取更精确的姿态角，需要安装**I2Cdevlib**库：

```cpp
/*
 * ============================================================
 * 3轴手势跟随机械臂 — DMP版本
 * 使用MPU6050内置DMP输出欧拉角
 * ============================================================
 * 依赖库：
 *   1. I2Cdev (https://github.com/jrowberg/i2cdevlib)
 *   2. MPU6050 (I2Cdevlib子库)
 * ============================================================
 */

#include "I2Cdev.h"
#include "MPU6050_6Axis_MotionApps20.h"
#include <Servo.h>

// ====== MPU6050和DMP变量 ======
MPU6050 mpu;
bool dmpReady = false;
uint8_t devStatus;
uint16_t packetSize;
uint8_t fifoBuffer[64];

Quaternion q;
VectorFloat gravity;
float ypr[3];  // yaw, pitch, roll (弧度)

// ====== 舵机 ======
Servo servoBase, servoShoulder, servoElbow;
float currentServo[3] = {90, 90, 90};
#define SMOOTH  0.15f

// ====== DMP中断处理 ======
volatile bool mpuInterrupt = false;
void dmpDataReady() {
    mpuInterrupt = true;
}

// ====== 初始化 ======
void setup() {
    Serial.begin(115200);
    
    // 初始化I2C
    #if I2CDEV_IMPLEMENTATION == I2CDEV_ARDUINO_WIRE
        Wire.begin();
        Wire.setClock(400000);
    #endif
    
    mpu.initialize();
    
    if (!mpu.testConnection()) {
        Serial.println("MPU6050连接失败！");
        while (1);
    }
    
    // 初始化DMP
    devStatus = mpu.dmpInitialize();
    
    // 设置偏移（请根据实际校准结果修改）
    mpu.setXGyroOffset(220);
    mpu.setYGyroOffset(76);
    mpu.setZGyroOffset(-85);
    mpu.setZAccelOffset(1688);
    
    if (devStatus == 0) {
        mpu.CalibrateAccel(6);
        mpu.CalibrateGyro(6);
        mpu.setDMPEnabled(true);
        
        // 配置中断（可选，使用INT引脚）
        attachInterrupt(0, dmpDataReady, RISING);  // D2引脚
        
        packetSize = mpu.dmpGetFIFOPacketSize();
        dmpReady = true;
        Serial.println("DMP初始化成功！");
    } else {
        Serial.print("DMP初始化失败，错误码: ");
        Serial.println(devStatus);
    }
    
    // 初始化舵机
    servoBase.attach(9);
    servoShoulder.attach(10);
    servoElbow.attach(11);
    servoBase.write(90);
    servoShoulder.write(90);
    servoElbow.write(90);
}

// ====== 主循环 ======
void loop() {
    if (!dmpReady) return;
    
    // 等待DMP数据就绪
    if (!mpu.dmpGetCurrentFIFOPacket(fifoBuffer)) return;
    
    // 解析DMP输出
    mpu.dmpGetQuaternion(&q, fifoBuffer);
    mpu.dmpGetGravity(&gravity, &q);
    mpu.dmpGetYawPitchRoll(ypr, &q, &gravity);
    
    // 弧度转角度
    float yaw   = ypr[0] * 180.0 / M_PI;
    float pitch = ypr[1] * 180.0 / M_PI;
    float roll  = ypr[2] * 180.0 / M_PI;
    
    // 映射到舵机
    float tBase     = constrain(90 + yaw,   0, 180);
    float tShoulder = constrain(90 + pitch, 0, 180);
    float tElbow    = constrain(90 + roll,  0, 180);
    
    currentServo[0] += SMOOTH * (tBase - currentServo[0]);
    currentServo[1] += SMOOTH * (tShoulder - currentServo[1]);
    currentServo[2] += SMOOTH * (tElbow - currentServo[2]);
    
    servoBase.write((int)currentServo[0]);
    servoShoulder.write((int)currentServo[1]);
    servoElbow.write((int)currentServo[2]);
    
    // 调试输出
    Serial.print("Yaw:"); Serial.print(yaw, 1);
    Serial.print(" Pitch:"); Serial.print(pitch, 1);
    Serial.print(" Roll:"); Serial.println(roll, 1);
}
```

### 10.5 电路注意事项

**电源设计**：

- Arduino的5V引脚最大输出电流约400-500mA
- 单个SG90舵机堵转电流可达500mA+，**严禁直接从Arduino供电**
- 必须使用独立5V电源（如LM7805稳压模块或USB充电器），Arduino与舵机电源**共地**

```
+----------+        +------------------+        +---------+
| 5V电源   |------->| 舵机供电总线      |------->| 3x SG90 |
| (2A+)    |        | (所有舵机红线)    |        |         |
+----------+        +------------------+        +---------+
    |                                               |
    GND (共地) <----------------------------------- GND
    |
+----------+                                      
| Arduino  |------ PWM D9/D10/D11 -----> 舵机信号线
| (逻辑控制)|                                      
+----------+                                      
    |
    +-- I2C SCL/SDA ---> MPU6050
```

**I2C上拉电阻**：

GY-521模块已内置4.7kΩ上拉电阻。若使用裸芯片或总线较长（>30cm），建议额外添加上拉电阻增强信号完整性。

## 11. 故障排查

### 11.1 常见问题诊断表

| 现象 | 可能原因 | 解决方案 |
|---|---|---|
| 串口打印"MPU6050 not found" | I2C接线错误；地址不匹配（AD0引脚） | 检查SDA/SCL接线；尝试0x69地址 |
| 所有读数为0或固定值 | 传感器处于睡眠模式 | 向PWR_MGMT_1(0x6B)写入0x01唤醒 |
| 数据剧烈跳动/噪声大 | 电源不稳；I2C线过长；无低通滤波 | 添加100nF退耦电容；缩短线缆；启用DLPF |
| 角度缓慢漂移 | 陀螺仪零点未校准 | 执行校准程序，存储偏移值 |
| Yaw角快速漂移 | 无磁力计辅助；正常物理现象 | 使用DMP减轻漂移；如需绝对Yaw需加磁力计 |
| DMP初始化返回错误码 | 传感器未平放；电源不稳 | 确保初始化时水平静止；检查供电 |
| DMP初始化返回错误码8 | 自检失败（传感器不平/振动） | 必须平放！检查是否有振动源 |
| 舵机不响应或抖动 | 供电不足；信号线接触不良 | 独立供电；检查信号线；添加平滑滤波 |
| 机械臂动作反向 | 传感器与舵机坐标系不对应 | 调整映射公式中的正负号 |
| 角度范围不够 | 传感器±90°映射不合理 | 调整sensorToServo中的偏移量和缩放比例 |

### 11.2 调试技巧

1. **分步验证法**：
   - 第一步：验证I2C通信（读取WHO_AM_I应返回0x68）
   - 第二步：验证原始数据读取（静止时Z轴加速度≈16384）
   - 第三步：验证校准（静止时三轴陀螺仪≈0）
   - 第四步：验证姿态解算（手动倾斜观察角度变化是否合理）
   - 第五步：验证舵机映射（观察角度与舵机位置是否对应）

2. **串口绘图器**：使用Arduino IDE的串口绘图器（Tools → Serial Plotter）可视化角度曲线，快速发现噪声或漂移问题。

3. **数据格式**：输出CSV格式便于后续分析：
   ```cpp
   Serial.print(roll); Serial.print(",");
   Serial.print(pitch); Serial.print(",");
   Serial.println(yaw);
   ```

### 11.3 性能优化建议

| 优化项 | 建议 | 预期效果 |
|---|---|---|
| I2C速率 | 设置为400kHz（Fast Mode） | 数据读取延迟从~3ms降至~1ms |
| 采样率 | 设置200~500Hz | 平滑跟踪，不过度占用CPU |
| 滤波系数 | 互补滤波alpha=0.94~0.98 | alpha越高越平滑，越低响应越快 |
| 平滑系数 | 舵机平滑=0.1~0.3 | 消除抖动，需平衡响应速度 |
| 死区设置 | 小角度（±3°）不响应 | 消除手部自然颤抖 |
| 预热 | 上电后等待2~3分钟再校准 | 减少温度漂移 |

## 附录A：姿态角定义

```
        Z（上）
         |
         |____ Y（前）
        /
       X（右）

Roll（横滚角）: 绕X轴旋转  → 左右倾斜
Pitch（俯仰角）: 绕Y轴旋转 → 前后俯仰
Yaw（偏航角）: 绕Z轴旋转   → 水平旋转
```

## 附录B：坐标系约定

MPU6050采用右手坐标系：

| 轴 | 正方向 | 对应姿态角 |
|---|---|---|
| X | 向右 | Roll（横滚） |
| Y | 向前 | Pitch（俯仰） |
| Z | 向下（重力方向） | Yaw不适用 |

> 注意：不同库/教程可能使用不同的坐标系约定，集成时请确认符号一致性。

## 附录C：换算公式速查

| 转换 | 公式 |
|---|---|
| 原始值 → 加速度(g) | `accel_g = raw / sensitivity` |
| 原始值 → 角速度(°/s) | `gyro_dps = raw / sensitivity` |
| 弧度 → 角度 | `deg = rad * 180 / PI` |
| 角度 → 弧度 | `rad = deg * PI / 180` |
| 互补滤波 | `angle = α*(angle+gyro*dt) + (1-α)*accel_angle` |
| 四元数→Roll | `atan2(2(w*x+y*z), 1-2(x²+y²))` |
| 四元数→Pitch | `asin(2(w*y-z*x))` |
| 四元数→Yaw | `atan2(2(w*z+x*y), 1-2(y²+z²))` |
